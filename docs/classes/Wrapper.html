Wrapper Class

Module: WebWorker

This documentation is for the worker script wrapper. Every user webworker script is wrapped within this shell. All properties and methods described here are attached to the 'self' object within the worker script.

_init

() private chainable

This method is called when initializing the worker script. It also triggers the WORKER_LOADED event on the base worker instance.

_main

() private chainable

The main worker script that is provided by the user is injected into this function. Also any arguments passed to the start method are passed on to this function.

_nativeClose

() private

The native worker close method is overwritten in the wrapper to give consistent behaviour with the WebWorker class. This method serves as an alias for the native version of the method to be used internally.

_removeListenerFromEventType

(
  • eventType
  • [listener]
)
private chainable

Removes an event listener for a particular event type. Optionally if the listener is not passed or is null it removes all listeners for the specified event type.

Parameters:

  • eventType String

    Event type for which the listener is being removed.

  • [listener] Function optional

    The listener that needs to be removed. If not passed in or passed as null all listeners for the specified event type are removed.

start

() chainable

This method is called by the base worker instance when it's start method is invoked. It also gets the arguments from the base worker. Internally it calls the Wrapper/main:method method with arguments.

terminate

(
  • [terminateNow]
)
chainable

Terminates the worker from within the worker script. Note that terminates from within the worker are asynchronous. This method calls the terminateHandler, if it is assigned and is a function, and then goes on to communicate with the base worker class to call the method terminateNow.

The WebWorker class does provide a synchronous way to terminate the worker but it is not supported from within the worker script itself.

Parameters:

  • [terminateNow] Boolean optional

    If set to true the worker also does a self.close() after it communicates the terminate status to the base worker instance. It is recommended to use the method terminateNow to achieve the same goal as it gives a more clear semantics in your code.

terminateHandler

()

This method is called by the script whenever the worker is made to terminate. By default this method does nothing but it exists so that you can override it with a custom function to possibly perform cleanup operations if the worker is being made to terminate.

terminateNow

()

Terminates the worker immediately from within the worker script. This method internally calls the terminate method to terminate the worker.

Note: Calling this method will NOT invoke the terminate handler.
